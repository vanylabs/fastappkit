# Changelog - Version 0.2.0

**Release Date**: 2025-12-31
**Status**: Alpha

## Added

### External Apps

-   Enhanced external app templates with improved configuration management
-   Added `config.py` template for external apps to handle settings independently
-   Added `.env.example` template for external apps to document required environment variables
-   Improved `main.py` template for external apps with better structure and configuration handling
-   Enhanced migration environment configuration for external apps to support independent development

### Configuration

-   Improved database URL handling with Pydantic Field configuration
-   Better environment variable management for external apps
-   Enhanced settings protocol for better type safety and validation

## Changed

### Python Version Support

-   Updated Python version constraints from `>=3.11,<3.13` to `>=3.11,<4.0` for better compatibility and future-proofing

### CLI

-   Enhanced `fastappkit app new --as-package` command with improved template generation
-   Updated development server CLI options and documentation

### Documentation

-   Updated configuration guides with improved examples and clarity
-   Enhanced deployment documentation with better CLI option descriptions
-   Improved API reference documentation
-   Updated CLI reference with clearer command descriptions

### Settings

-   Refactored settings to use `database_url` and `debug` with Pydantic's Field for better compatibility
-   Improved settings protocol implementation for better type safety

## Fixed

### External Apps (Issue #5)

-   Fixed external app template generation issues
-   Resolved migration environment configuration for external apps
-   Improved Alembic configuration for external apps

### Settings (Issue #3)

-   Fixed settings configuration and database URL handling
-   Improved compatibility between core projects and external apps
-   Enhanced migration context for better database connection handling

## Technical Details

This release focuses on improving external app support and configuration management. The changes enable better independent development of external apps while maintaining compatibility with the core fastappkit framework. The Python version constraint update allows for broader compatibility and future Python versions.
